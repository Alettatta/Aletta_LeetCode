# Problem Description
给定一个未排序的整数数组 nums，请你找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

数据范围：
0 <= nums.length <= 10^5
-10^9 <= nums[i] <= 10^9


# Key Points
需要遍历全部的元素，想哈希表方法（快速判断一个元素是否出现过）

哈希存储的数据结构：
- 数组（元素数量有限，例如字母a-z）
- set（元素数量未定，注意集合的元素不能重复）
- dict（map，注意确定好谁是key谁是value）


# Solution
因为不需要用到数组下标，可以用 set 存储，同时 set 不含重复元素，所以很清晰看到单个元素情况

怎么查找序列起点：-1不在


# Code

## LC version

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 需要遍历整个数组，找到序列的起点，然后不断+1看那个数字是否存在，存在则最长序列+1
        # 问题：如何找到序列的起点——前无古人，后有来者，-1不在+1在
        # 因为原始数组中元素的下标用不到，可以定义一个集合，使得元素连续

        hash = set(nums) # 集合元素不重复，那就是(0,100,101……)
        max_length = 0
        for num in hash:
            if num - 1 not in hash:
                start = num
                length = 1
                while start + 1 in hash:
                    start += 1
                    length += 1
                max_length = max(length, max_length)
        return max_length
```

## ACM version

**ACM 模式的注意点：**

- 需要 import 完整的类（包括 sys、typing 等）
- 数据在标准输入流 stdin 中，全部是原始的文本字符串
- 必须用 print() 手动将结果写到标准输出流 stdout
- 需要写 while 或 for line in sys.stdin 循环处理，直到文件结束（EOF）

```python

from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 需要遍历整个数组，找到序列的起点，然后不断+1看那个数字是否存在，存在则最长序列+1
        # 问题：如何找到序列的起点——前无古人，后有来者，-1不在+1在
        # 因为原始数组中元素的下标用不到，可以定义一个集合，使得元素连续

        hash = set(nums) # 集合元素不重复，那就是(0,100,101……)
        max_length = 0
        for num in hash:
            if num - 1 not in hash:
                start = num
                length = 1
                while start + 1 in hash:
                    start += 1
                    length += 1
                max_length = max(length, max_length)

# -- core code template end


# ACM 输入输出
data = sys.stdin.read().split()
idx = 0
while idx < len(data):
    n = int(data[idx]); idx += 1
    nums = [int(x) for x in data[idx:idx + n]]; idx += n
    result = Solution().longestConsecutive(nums)

    print(result)
```
