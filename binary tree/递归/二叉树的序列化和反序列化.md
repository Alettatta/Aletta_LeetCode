# Problem Description

序列化是将一个数据结构或者对象转换为连续的比特位的操作，进而可以将转换后的数据存储在一个文件或者内存中，同时也可以通过网络传输到另一个计算机环境，采取相反方式重构得到原数据。

请设计一个算法来实现二叉树的序列化与反序列化。这里不限定你的序列 / 反序列化算法执行逻辑，你只需要保证一个二叉树可以被序列化为一个字符串并且将这个字符串反序列化为原始的树结构。

提示: 输入输出格式与 LeetCode 目前使用的方式一致，详情请参阅 LeetCode 序列化二叉树的格式。你并非必须采取这种方式，你也可以采用其他的方法解决这个问题。



# Key Points

## 1. 什么是"序列化"和"反序列化"

想象一下这个场景:你在电脑内存里有一个复杂的数据结构(比如一棵二叉树,或者一个对象),现在你想把它**存到硬盘上**,或者**通过网络发给另一台电脑**。

问题来了:内存中的数据结构(比如对象、指针、树的节点)是**依赖于编程语言和内存地址**的东西,没法直接"复制粘贴"到硬盘文件或者网络包里。

所以我们需要:

- **序列化(Serialize)**:把内存中的数据结构,转换成一个**字符串**(或字节流),这个字符串可以存文件、写数据库、通过网络传输。
- **反序列化(Deserialize)**:把这个字符串重新解析,还原出原来的数据结构。

一句话总结:
```
内存中的数据结构  --序列化-->  字符串（可存储/可传输）
字符串           --反序列化-->  内存中的数据结构
```

于是就有了几种通用的数据格式,JSON(JavaScript Object Notation)是最常用的一种。它长这样:


## 4. 回到数据结构:为什么二叉树也要序列化

如果你想把一棵二叉树:

```
      1
     / \
    2   3
       / \
      4   5
```

存到文件里,或者传给另一个程序,你不能直接存内存地址(节点对象、`left`/`right`指针这些东西,离开当前程序的内存空间就没有意义了)。

所以需要把树"压扁"成一个字符串,比如:

```
"1,2,#,#,3,4,#,#,5,#,#"
```

（这里 `#` 表示空节点,用来在反序列化时区分空节点的位置）

这个字符串就可以存文件、发网络。对方拿到这个字符串后,按照约定的规则重新"解析"出一棵一模一样的二叉树。这个过程就是:

- **序列化**:树 → 字符串
- **反序列化**:字符串 → 树



# Solution

当二叉树中节点的值不存在重复时：

如果你的序列化结果中不包含空指针的信息，且你只给出一种遍历顺序，那么你无法还原出唯一的一棵二叉树。

如果你的序列化结果中不包含空指针的信息，且你会给出两种遍历顺序，分两种情况：

1. 如果你给出的是前序和中序，或者后序和中序，那么你可以还原出唯一的一棵二叉树。

2. 如果你给出前序和后序，那么你无法还原出唯一的一棵二叉树。

如果你的序列化结果中**包含空指针的信息，且你只给出一种遍历顺序**，也要分两种情况：

1.  **如果你给出的是前序或者后序，那么你可以还原出唯一的一棵二叉树**。

2. 如果你给出的是中序，那么你无法还原出唯一的一棵二叉树。（因为反序列的时候未知 root）

下面就使用 包含空指针 null + 前序遍历方法 进行序列化和反序列化，前序遍历方法如下：

```python
res = []

def traverse(root):
    if root is None: 
        res.append(-1) # 暂且用数字 -1 代表空指针 null
        return

    # ****** 前序位置 ******
    res.append(root.val)
    traverse(root.left)
    traverse(root.right)
```

问题很明显:build 函数缺少递归终止条件(base case)。

问题分析
递归函数 build 每次都无条件执行:

python
rootVal = preorder[preStart]
但是当子树为空时(比如某个节点没有左子树或右子树),对应的区间会变成无效区间,即 preStart > preEnd(或 inStart > inEnd)。这时候应该返回 None,但代码里没有做这个判断,于是继续往下执行,导致:

preorder[preStart] 数组越界或取到错误的值
递归无法终止,一直调用下去,最终报错(你看到的 preorder[preStart] 报错就是这个原因)
举例说明
假设某个节点没有左子树,那么调用左子树时:

python
root.left = self.build(preorder, preStart+1, preStart+leftSide, ...)
如果 leftSide == 0(没有左子树),那么区间是 preStart+1 到 preStart+0,也就是 preStart+1 到 preStart,这是一个空区间(start > end)。

但函数没有检查这个情况,直接执行 preorder[preStart],这时候 preStart 实际上已经超出了正确范围,取到了错误的元素,基于错误的值继续递归构建左子树,导致递归停不下来(或者干脆数组越界)。

修复方法
在 build 函数开头加上终止条件:

python
def build(self, preorder: List[int], preStart: int, preEnd: int,
         inorder: List[int], inStart: int, inEnd: int) -> Optional[TreeNode]:
    
    # 加上这个判断！区间无效(空)，说明子树不存在
    if preStart > preEnd:
        return None
    
    rootVal = preorder[preStart]
    index = inorder.index(rootVal)
    leftSide = index - inStart
    root = TreeNode(rootVal)
    
    root.left = self.build(preorder, preStart+1, preStart+leftSide,
                          inorder, inStart, index-1)
    root.right = self.build(preorder, preStart+leftSide+1, preEnd,
                           inorder, index+1, inEnd)
    
    return root
为什么加 preStart > preEnd 就够了
因为 preorder 数组和 inorder 数组的区间长度应该是一一对应的(左子树在两个数组里对应的元素个数相同,右子树也是),只要检查其中一个数组的区间是否有效即可,不需要两个都判断。

加上这一行之后,当遇到空子树时函数会立刻返回 None,递归自然终止,就不会出现数组越界或死循环的问题了。
