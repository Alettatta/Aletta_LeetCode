# Problem Description

序列化是将一个数据结构或者对象转换为连续的比特位的操作，进而可以将转换后的数据存储在一个文件或者内存中，同时也可以通过网络传输到另一个计算机环境，采取相反方式重构得到原数据。

请设计一个算法来实现二叉树的序列化与反序列化。这里不限定你的序列 / 反序列化算法执行逻辑，你只需要保证一个二叉树可以被序列化为一个字符串并且将这个字符串反序列化为原始的树结构。

提示: 输入输出格式与 LeetCode 目前使用的方式一致，详情请参阅 LeetCode 序列化二叉树的格式。你并非必须采取这种方式，你也可以采用其他的方法解决这个问题。



# Key Points

## 1. 什么是"序列化"和"反序列化"

想象一下这个场景:你在电脑内存里有一个复杂的数据结构(比如一棵二叉树,或者一个对象),现在你想把它**存到硬盘上**,或者**通过网络发给另一台电脑**。

问题来了:内存中的数据结构(比如对象、指针、树的节点)是**依赖于编程语言和内存地址**的东西,没法直接"复制粘贴"到硬盘文件或者网络包里。

所以我们需要:

- **序列化(Serialize)**:把内存中的数据结构,转换成一个**字符串**(或字节流),这个字符串可以存文件、写数据库、通过网络传输。
- **反序列化(Deserialize)**:把这个字符串重新解析,还原出原来的数据结构。

一句话总结:
```
内存中的数据结构  --序列化-->  字符串（可存储/可传输）
字符串           --反序列化-->  内存中的数据结构
```

## 2. 为什么需要 JSON 这种格式

序列化后的字符串,总得有个"规则"吧?不能随便写,不然对方怎么知道怎么解析回去?

于是就有了几种通用的数据格式,JSON(JavaScript Object Notation)是最常用的一种。它长这样:

```json
{
  "name": "小明",
  "age": 18,
  "hobbies": ["篮球", "编程"],
  "address": {
    "city": "北京",
    "zip": "100000"
  }
}
```

它只支持几种简单类型:
- 字符串 `"abc"`
- 数字 `123`
- 布尔值 `true` / `false`
- 数组 `[1, 2, 3]`
- 对象(键值对) `{"key": "value"}`
- `null`

关键点:JSON 是纯文本,不管你用 Python、Java、Go、JavaScript,大家都能读懂这个格式。这就是"独立于编程语言"的含义。

## 3. 举例:序列化一个 Python 对象

假设你有一个 Python 的类:

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("小明", 18)
```

`p` 这个对象存在内存里,你没法直接把它存到文件或者发给别人(不同语言根本读不懂 Python 对象的内存结构)。

**序列化**成 JSON 字符串:
```python
import json
data = json.dumps({"name": p.name, "age": p.age})
print(data)  
# 输出: '{"name": "\u5c0f\u660e", "age": 18}'
```

现在 `data` 是一个**字符串**,可以存文件、发网络请求。

对方(可能是 Java 服务器、前端 JavaScript)拿到这个字符串后,反序列化还原出数据:

```python
# 反序列化
obj = json.loads(data)
print(obj)  # {'name': '小明', 'age': 18}
```

或者前端 JavaScript 直接:
```javascript
let obj = JSON.parse(data);
console.log(obj.name); // "小明"
```

看,同一段 JSON 字符串,Python 和 JavaScript 都能解析,这就是"独立于编程语言"的意义。

## 4. 回到数据结构:为什么二叉树也要序列化

如果你想把一棵二叉树:

```
      1
     / \
    2   3
       / \
      4   5
```

存到文件里,或者传给另一个程序,你不能直接存内存地址(节点对象、`left`/`right`指针这些东西,离开当前程序的内存空间就没有意义了)。

所以需要把树"压扁"成一个字符串,比如:

```
"1,2,#,#,3,4,#,#,5,#,#"
```

（这里 `#` 表示空节点,用来在反序列化时区分空节点的位置）

这个字符串就可以存文件、发网络。对方拿到这个字符串后,按照约定的规则重新"解析"出一棵一模一样的二叉树。这个过程就是:

- **序列化**:树 → 字符串
- **反序列化**:字符串 → 树



# Solution

当二叉树中节点的值不存在重复时：

如果你的序列化结果中不包含空指针的信息，且你只给出一种遍历顺序，那么你无法还原出唯一的一棵二叉树。

如果你的序列化结果中不包含空指针的信息，且你会给出两种遍历顺序，分两种情况：

1. 如果你给出的是前序和中序，或者后序和中序，那么你可以还原出唯一的一棵二叉树。

2. 如果你给出前序和后序，那么你无法还原出唯一的一棵二叉树。

如果你的序列化结果中**包含空指针的信息，且你只给出一种遍历顺序**，也要分两种情况：

1.  **如果你给出的是前序或者后序，那么你可以还原出唯一的一棵二叉树**。

2. 如果你给出的是中序，那么你无法还原出唯一的一棵二叉树。（因为反序列的时候未知 root）

下面就使用 包含空指针 null + 前序遍历方法 进行序列化和反序列化，前序遍历方法如下：

```python
res = []

def traverse(root):
    if root is None: 
        res.append(-1) # 暂且用数字 -1 代表空指针 null
        return

    # ****** 前序位置 ******
    res.append(root.val)
    traverse(root.left)
    traverse(root.right)
```

```python
class Codec:
    # 因为序列化和反序列化都要用到数组，需要空指针#和间隔符号,
    SEP = ','
    NULL = '#'

    def serialize(self, root):
        sb = [] # 我们先建立一个数组，方便拼接字符串
        self._serialize(root,sb)
        return "".join(sb)
    
    def _serialize(self, root, sb):
        if root is None:
            sb.append(self.NULL)
            sb.append(self.SEP)
            return
        
        sb.append(str(root.val))
        sb.append(self.SEP)

        self._serialize(root.left, sb)
        self._serialize(root.right, sb)

        

    def deserialize(self, data): # 本质是字符串转换为列表
        nodes = data.split(self.SEP)
        return self._deserialize(nodes)
    
    def _deserialize(self, nodes):
        if not nodes:
            return None
        
        first = nodes.pop(0)
        if first == self.NULL:
            return None
        root = TreeNode(int(first))

        root.left = self._deserialize(nodes)
        root.right = self._deserialize(nodes)

        return root
```

注意：
1. None是Python 的一个特殊值,表示"空/没有"；self.NULL (也就是字符串 '#')表示树里"这个节点不存在"(root is None)
2. popleft() 是 collections.deque(双端队列) 才有的方法,不是普通 list 的方法————当你 pop(0) 删除第一个元素时,后面所有元素都要往前挪一位,这是 O(n) 的操作。如果你反复执行 pop(0)(比如树有 n 个节点,要执行 n 次),总的时间复杂度会退化成 O(n²),当 n 很大时会很慢。
   
   我们可以把代码改成：
```python
from collections import deque

def deserialize(self, data):
    nodes = deque(data.split(self.SEP))  # 换成 deque
    return self._deserialize(nodes)

def _deserialize(self, nodes):
    if not nodes:
        return None
    
    first = nodes.popleft()  # O(1)，比 pop(0) 快很多
    if first == self.NULL:
        return None
    root = TreeNode(int(first))
    root.left = self._deserialize(nodes)
    root.right = self._deserialize(nodes)
    return root
```
    


# Complexity Analysis
- 时间复杂度：O(n)
- 空间复杂度：O(1)
