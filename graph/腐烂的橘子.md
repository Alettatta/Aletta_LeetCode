https://labuladong.online/zh/problem/leetcode/rotting-oranges/description/


# Problem Description
在一个 m x n 的网格 grid 中，每个单元格可以有以下三个值之一：

值 0 代表空单元格；

值 1 代表新鲜橘子；

值 2 代表腐烂的橘子。

每分钟，腐烂的橘子周围 4 个方向上相邻的新鲜橘子都会腐烂。

返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 -1。


# Solution
"岛屿数量"里，每次找到一个新的 '1' 就单独做一次 DFS，是因为每个岛屿是独立的，互不影响，可以逐个"数"出来。

但这道题不一样：所有腐烂橘子是同时、同步地向外扩散腐烂的，是"一分钟"这个时间维度上大家一起走一步。如果你像你写的伪代码那样，对每个腐烂橘子 grid[i][j]==2 单独跑一次 BFS，会有两个致命问题：

时间会被重复计算、互相干扰。比如橘子 A 和橘子 B 分别是两个腐烂源，中间有一个新鲜橘子 C，C 到 A 距离是 3，到 B 距离是 1。如果你先对 A 做一次完整 BFS 把 C 腐烂掉（记录时间 3），再对 B 做 BFS 时，C 已经是 '2' 了，不会再被处理，你就漏算了"实际上 C 应该 1 分钟就烂了"这个更短的时间。
这道题要求的是"整体全部腐烂完的时间"，即所有橘子里最晚被腐烂的那个的时间，而不是把每个源头单独跑一遍再求和或者求最大值——这样做语义就错了。

正确思路是：把所有初始腐烂的橘子，同时作为起点，一起放入一个队列，做一次"多源 BFS"（multi-source BFS）。这样每一层（每一分钟）所有腐烂橘子同时向外扩散一步，天然保证了"最短时间腐烂"的正确性，也不会有上面说的冲突问题。


# Code

## LC version

```python
from collections import deque

class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        # 因为是一层一层往外面腐烂，所以是BFS算法，同时记录step
        # 因为防止反复腐烂，要记录开始所有腐烂源，然后往外面扩散

        q = deque()
        m, n = len(grid), len(grid[0])
        for i in range(m):
            for j in range(n):
                if grid[i][j] == 2:
                    q.append((i,j))

         # 方向数组，方便计算上下左右的坐标
        dirs = [(-1, 0), (1, 0), (0, -1), (0, 1)]

        # BFS
        step = 0
        while q:
            sz = len(q)
            for _ in range(sz):
                cur = q.popleft()
                for dir in dirs:
                    x, y = cur[0] + dir[0], cur[1] + dir[1]
                    if 0 <= x < m and 0 <= y < n and grid[x][y] == 1:
                        grid[x][y] = 2
                        q.append((x, y))
            step += 1 # 扩散步数加一

        # 检查还有无新鲜橘子
        for i in range(m):
            for j in range(n):
                if grid[i][j] == 1:
                    return -1

        return step - 1 if step else 0
        # 注意，BFS 扩散的步数需要减一才是最终结果，你可以用最简单的情况，比方说只有一个腐烂橘子的情况验证一下
```

## ACM version

**ACM 模式的注意点：**

- 需要 import 完整的类（包括 sys、typing 等）
- 数据在标准输入流 stdin 中，全部是原始的文本字符串
- 必须用 print() 手动将结果写到标准输出流 stdout
- 需要写 while 或 for line in sys.stdin 循环处理，直到文件结束（EOF）

from collections import deque
from typing import List

        dirs = [(-1, 0), (1, 0), (0, -1), (0, 1)]

        step = 0

            for _ in range(sz):

m = 0
n = 0
grid = []
rows_left = 0
for line in sys.stdin:
    parts = line.split()
    if not parts:
        continue
    if rows_left == 0:
        m, n = int(parts[0]), int(parts[1])
        grid = []
        rows_left = m
    else:
        grid.append(list(map(int, parts)))
        rows_left -= 1
        if rows_left == 0:
            result = Solution().orangesRotting(grid)
            print(result)
