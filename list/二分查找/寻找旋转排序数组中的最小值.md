# Problem Description
已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：

若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]

注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]]。

给你一个元素值 互不相同 的数组 nums，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。


# Key Points
一开始我写的代码是这样的：
```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        # 其实就是要找断点下面的数字——注意返回的是值
        left, right = 0, len(nums) - 1

        while left <= right:
            mid = (left + right) // 2    
                
            if nums[mid] >= nums[left]:# 说明断点在mid后面
                left = mid + 1
            else: # 断点在mid之前
                right = mid -1
                
        return nums[left]
```

结果报错left指针越界了，原因如下：

[33]中有target约束，即使 mid == left 时条件成立，也能通过判断 target 的位置来正确移动指针。

```python
if nums[mid] >= nums[left]:
    # 左半部分有序
    if nums[left] <= target and nums[mid] > target:
        right = mid - 1  # target 在左半部分
    else:
        left = mid + 1   # target 在右半部分
```

这道题以完全有序数组[1,2,3,4,5]为例：当 mid == left 时，nums[mid] >= nums[left] 必然成立，导致 left 不断右移，最终越界。

```python
left=0, right=4
第1轮: mid=2, nums[mid]=3 >= nums[left]=1 ✓
       left = mid + 1 = 3

left=3, right=4
第2轮: mid=3, nums[mid]=4 >= nums[left]=4 ✓  # 注意：相等！
       left = mid + 1 = 4

left=4, right=4
第3轮: mid=4, nums[mid]=5 >= nums[left]=5 ✓  # 又相等！
       left = mid + 1 = 5

退出循环: left=5, right=4
return nums[5]  # ❌ IndexError!
```

**所以在定位mid之后首先要做：万一一下子就找到了呢**——判断断点处数学关系

```python
if nums[mid] > nums[mid + 1]:
  return nums[mid + 1]
if nums[mid - 1] > nums[mid]:
  return nums[mid]
```

# Code

## LC version

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        # 其实就是要找断点下面的数字——注意返回的是值
        left, right = 0, len(nums) - 1

        # 如果数组没有旋转，直接返回，比如[1,2,3,4]
        if nums[left] <= nums[right]:
            return nums[left]
            
        while left <= right:
            mid = (left + right) // 2    
            
            # 检查 mid 是否是最小值：断点处的数学规律
            if nums[mid] > nums[mid + 1]:
                return nums[mid + 1]
            if nums[mid - 1] > nums[mid]:
                return nums[mid]
                
            if nums[mid] >= nums[left]:# 说明断点在mid后面
                left = mid + 1
            else: # 断点在mid之前
                right = mid -1
                
        return nums[left]
```

## ACM version

**ACM 模式的注意点：**

- 需要 import 完整的类（包括 sys、typing 等）
- 数据在标准输入流 stdin 中，全部是原始的文本字符串
- 必须用 print() 手动将结果写到标准输出流 stdout
- 需要写 while 或 for line in sys.stdin 循环处理，直到文件结束（EOF）

```python
import sys
from typing import List

class Solution:
    def findMin(self, nums: List[int]) -> int:
        # 其实就是要找断点下面的数字——注意返回的是值
        left, right = 0, len(nums) - 1

        # 如果数组没有旋转，直接返回，比如[1,2,3,4]
        if nums[left] <= nums[right]:
            return nums[left]
            
        while left <= right:
            mid = (left + right) // 2    
            
            # 检查 mid 是否是最小值：断点处的数学规律
            if nums[mid] > nums[mid + 1]:
                return nums[mid + 1]
            if nums[mid - 1] > nums[mid]:
                return nums[mid]
                
            if nums[mid] >= nums[left]:# 说明断点在mid后面
                left = mid + 1
            else: # 断点在mid之前
                right = mid -1
                
        return nums[left]

for line in sys.stdin:
    n = int(line.strip())
    nums = list(map(int, sys.stdin.readline().split()))
    result = Solution().findMin(nums)

    print(result)
```
